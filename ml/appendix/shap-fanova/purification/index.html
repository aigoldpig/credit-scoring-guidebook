    3. Purification — 트리 모형의 사후 정제¶

    작성 중

    이 페이지는 현재 작성 중입니다.

    왜 Purification이 필요한가¶

    트리 기반 모형(XGBoost, EBM 등)은 학습 과정에서 fANOVA의 직교 제약을 고려하지 않는다. 트리는 단지 예측 오차를 줄이는 방향으로 분할할 뿐이고, "이 효과가 main에 담겨야 하는가, interaction에 담겨야 하는가"를 구분하지 않는다.

    그 결과, 학습된 shape function에서 main effect와 교호작용이 뒤섞인다. 본래 \(f_1(X_1)\)에 있어야 할 효과가 \(f_{12}(X_1, X_2)\)에 누출되거나, 반대로 교호작용의 일부가 main effect에 흡수된다.

    1절 Non-identifiability에서 확인한 것처럼, 이 누출은 예측값을 전혀 바꾸지 않는다. 예측 성능은 동일하지만 해석이 완전히 달라진다. 이것을 사후적으로(post-hoc) 바로잡는 것이 purification(정제)이다.

    핵심 아이디어 — 숫자로 먼저 이해하기¶

    추상적 알고리즘을 보기 전에, 구체적인 숫자 예시로 직관을 잡자.

    설정¶

    신용평가 모형에서 DTI(부채비율)와 소득, 두 변수가 각각 2구간(Low, High)이다. XGBoost depth-2가 학습한 결과가 다음과 같다고 하자:

    component 값
    \(f_0\) (intercept) 0
    \(f_{\text{DTI}}(\text{Low})\) 0
    \(f_{\text{DTI}}(\text{High})\) 0
    \(f_{\text{소득}}(\text{Low})\) 0
    \(f_{\text{소득}}(\text{High})\) 0
    \(f_{12}\) 교호작용 소득=Low 소득=High
    DTI=Low +0.1 -0.7
    DTI=High +0.5 +0.1

    main effect가 전부 0이고, 모든 효과가 교호작용에 담겨 있다. 이 모형의 해석은 "DTI도 소득도 개별 효과는 없고, 오직 두 변수의 조합만 중요하다"가 된다.

    정말 그런가? 교호작용 테이블을 자세히 보면, 소득이 Low인 열의 값들(+0.1, +0.5)이 소득이 High인 열의 값들(-0.7, +0.1)보다 전반적으로 높다. 이것은 "소득이 낮으면 위험하다"는 소득의 main effect가 교호작용에 누출되어 있다는 신호다.

    Purification 실행¶

    1단계 — 소득의 누출을 찾아서 되돌린다

    교호작용 테이블에서 각 소득 구간별로, DTI 방향의 가중평균을 구한다 (균등 가중):

    \[ \text{소득=Low 열의 평균}: \frac{0.1 + 0.5}{2} = +0.3 \]
    \[ \text{소득=High 열의 평균}: \frac{-0.7 + 0.1}{2} = -0.3 \]

    이 평균값(+0.3, -0.3)은 소득 구간에만 의존하고 DTI와 무관하다. 따라서 교호작용이 아니라 소득의 main effect다. 이것을 교호작용에서 빼고, main effect로 옮긴다:

    \(f_{\text{소득}}\) 업데이트:

    이전 이동량 이후
    소득=Low 0 +0.3 +0.3
    소득=High 0 -0.3 -0.3

    \(f_{12}\) 업데이트 (각 열에서 해당 평균을 뺀다):

    \(f_{12}\) 소득=Low 소득=High
    DTI=Low 0.1 - 0.3 = -0.2 -0.7 - (-0.3) = -0.4
    DTI=High 0.5 - 0.3 = +0.2 0.1 - (-0.3) = +0.4

    검산: 소득=Low 열의 평균 = (-0.2 + 0.2)/2 = 0 ✓, 소득=High 열의 평균 = (-0.4 + 0.4)/2 = 0 ✓

    소득 방향의 누출이 제거되었다.

    2단계 — DTI의 누출을 찾아서 되돌린다

    이번에는 각 DTI 구간별로, 소득 방향의 가중평균을 구한다:

    \[ \text{DTI=Low 행의 평균}: \frac{-0.2 + (-0.4)}{2} = -0.3 \]
    \[ \text{DTI=High 행의 평균}: \frac{0.2 + 0.4}{2} = +0.3 \]

    이것은 DTI 구간에만 의존하는 효과 → DTI의 main effect다. 교호작용에서 빼고, main effect로 옮긴다:

    \(f_{\text{DTI}}\) 업데이트:

    이전 이동량 이후
    DTI=Low 0 -0.3 -0.3
    DTI=High 0 +0.3 +0.3

    \(f_{12}\) 업데이트 (각 행에서 해당 평균을 뺀다):

    \(f_{12}\) 소득=Low 소득=High
    DTI=Low -0.2 - (-0.3) = +0.1 -0.4 - (-0.3) = -0.1
    DTI=High 0.2 - 0.3 = -0.1 0.4 - 0.3 = +0.1

    검산: 모든 행 평균 = 0 ✓, 모든 열 평균 = 0 ✓ → 수렴 완료.

    결과 비교¶

    Purification 전:

    component 값 해석
    \(f_{\text{DTI}}\) Low=0, High=0 "DTI는 개별 효과 없음"
    \(f_{\text{소득}}\) Low=0, High=0 "소득도 개별 효과 없음"
    \(f_{12}\) ±0.1 ~ ±0.7 (큰 범위) "교호작용이 지배적"

    Purification 후:

    component 값 해석
    \(f_{\text{DTI}}\) Low=-0.3, High=+0.3 "DTI 높으면 위험 ↑"
    \(f_{\text{소득}}\) Low=+0.3, High=-0.3 "소득 높으면 위험 ↓"
    \(f_{12}\) ±0.1 (작은 범위) "순수 교호작용은 미미"

    같은 모형, 같은 예측값인데 해석이 완전히 달라졌다. 교호작용이 지배적이었던 모형이 실은 main effect가 핵심이었고, 교호작용은 미미했다. 이것이 purification의 효과다.

    예측값이 보존되는지 검산¶

    입력 Purification 전 Purification 후
    DTI=Low, 소득=High \(0 + 0 + 0 + (-0.7) = \mathbf{-0.7}\) \(0 + (-0.3) + (-0.3) + (-0.1) = \mathbf{-0.7}\) ✓
    DTI=High, 소득=Low \(0 + 0 + 0 + 0.5 = \mathbf{0.5}\) \(0 + 0.3 + 0.3 + (-0.1) = \mathbf{0.5}\) ✓
    DTI=Low, 소득=Low \(0 + 0 + 0 + 0.1 = \mathbf{0.1}\) \(0 + (-0.3) + 0.3 + 0.1 = \mathbf{0.1}\) ✓
    DTI=High, 소득=High \(0 + 0 + 0 + 0.1 = \mathbf{0.1}\) \(0 + 0.3 + (-0.3) + 0.1 = \mathbf{0.1}\) ✓

    4개 입력 모두 예측값이 정확히 일치한다. Purification은 효과의 귀속(attribution)만 바꾸고, 예측은 건드리지 않는다.

    Mass-Moving 알고리즘 (Lengerich et al., 2020)¶

    위 예시에서 한 것을 일반화한 것이 Mass-Moving 알고리즘이다. piecewise-constant 함수(트리 모형의 출력)에 대해 적용 가능하다.

    단일 텐서 정제 (Purify-Matrix)¶

    하나의 교호작용 텐서 \(T_u\)를 정제하는 절차:

    입력: 교호작용 텐서 T_u, 가중치 w

반복 (모든 slice 가중평균이 0이 될 때까지):

    for each 변수 i ∈ u:
        for each (나머지 변수 고정값) x_{u\i}:

            ① m ← 변수 i 방향으로 slice 가중평균 계산
            ② T_u 에서 m을 뺀다      ← 고차항에서 빼기
            ③ T_{u\i} 에 m을 더한다   ← 저차항에 더하기

    위 예시에서 ①②③이 정확히 어떻게 대응되는지 되짚어보자:

    단계 알고리즘 예시에서 한 것
    ① slice 가중평균 계산 "소득=Low 열의 DTI 방향 평균 = +0.3"
    ② 고차항에서 빼기 "소득=Low 열의 모든 값에서 0.3을 뺀다"
    ③ 저차항에 더하기 "\(f_{\text{소득}}(\text{Low})\)에 0.3을 더한다"

    전체 모형 정제 (Purify)¶

    실제 모형에는 2차뿐 아니라 3차 이상의 교호작용도 있을 수 있다. 이때는 가장 고차부터 내려온다:

    Purify (전체 모형):
    3차 interaction → 2차 interaction + main effect로 mass 이동
    2차 interaction → main effect로 mass 이동
    main effect → intercept(f₀)로 mass 이동

    각 단계에서 "이 텐서에 담긴 효과 중, 더 적은 수의 변수로 설명 가능한 부분"을 아래로 내려보낸다. 최종적으로 각 component에는 해당 변수 조합에서만 나타나는 순수한 효과만 남는다.

    수렴 보장¶

    위 예시에서는 2단계만에 모든 slice 평균이 0이 되었다. 항상 이렇게 빨리 끝나는 것인가?

    Theorem 1: 균등 가중이면 1회 반복¶

    모든 bin의 가중치(샘플 비율)가 균등하면, 각 변수를 한 번씩만 순회하면 수렴한다.

    위 예시가 정확히 이 경우다. DTI의 Low/High가 각각 50%, 소득의 Low/High도 각각 50% → 균등 가중 → 변수 2개를 한 바퀴 돌면 끝.

    Theorem 2: 일반 가중이면 \(O(\log(1/\varepsilon))\) 반복¶

    실제 데이터에서는 구간별 샘플 수가 다르다. 예를 들어 DTI=Low에 80%, DTI=High에 20%의 샘플이 몰려 있을 수 있다. 이 경우 한 번의 순회로는 slice 평균이 정확히 0이 되지 않는다.

    그러나 매 반복마다 정제되지 않은 잔여 mass가 최소 절반으로 줄어든다:

    \[ \|r^{(t+1)}\| \leq \frac{1}{2} \|r^{(t)}\| \]

    따라서:

    • 5회 반복 → 잔여 mass가 원래의 \(1/2^5 \approx 3\%\) 이하
    • 10회 반복 → 잔여 mass가 원래의 \(1/2^{10} \approx 0.1\%\) 이하

    실무적으로 5~10회 반복이면 충분하다.

    구현

    이 알고리즘은 Python 100줄 이내로 구현 가능하며, InterpretML에 내장되어 있다. GAM Purification 별도 라이브러리도 있다.

    Purification이 실제로 무엇을 바꾸는가 — 실증 사례¶

    사례 1: California Housing (위도 × 경도)¶

    캘리포니아 주택 가격 데이터셋(scikit-learn 내장)으로 XGBoost depth-2 모형을 학습한다. 변수 중에 위도(\(X_1\))와 경도(\(X_2\))가 있다. depth-2 트리는 두 변수의 교호작용을 자동으로 학습한다.

    학습된 모형의 shape function을 꺼내서 \(f_{\text{위도}}(X_1)\), \(f_{\text{경도}}(X_2)\), \(f_{\text{위도,경도}}(X_1, X_2)\)를 그려보면:

    Purification 전 — 무엇이 잘못되었나:

    캘리포니아에는 "해안에 가까울수록 비싸다"는 강한 패턴이 있다. 이것은 위도와 경도 각각의 main effect다:

    • 경도가 서쪽(해안 쪽)일수록 비싸다 → \(f_{\text{경도}}\)에 담겨야 할 효과
    • 위도가 SF·LA 위치일수록 비싸다 → \(f_{\text{위도}}\)에 담겨야 할 효과

    그런데 XGBoost는 학습 과정에서 이 구분을 신경 쓰지 않는다. depth-2 트리가 "경도가 서쪽 AND 위도가 SF 근처"를 하나의 leaf에서 같이 학습하면, 해안가 효과가 interaction 항 \(f_{12}\)에 누출된다.

    결과: interaction heatmap에서 가장 높은 값이 태평양 바다 위 좌표에 나타난다. 실제 주택이 없는 바다 한가운데가 "가장 비싼 지역"으로 표시되는 것이다. 이것은 main effect(해안 = 비쌈)가 interaction에 섞여 들어간 결과다.

    Purification 후 — 무엇이 바뀌는가:

    Purification은 \(f_{12}\)의 각 slice 가중평균을 계산해서 main effect로 되돌려보낸다.

    • "경도를 고정하고 위도에 대해 평균"을 내면 → 그 평균값은 경도의 main effect에 해당 → \(f_{\text{경도}}\)로 이동
    • "위도를 고정하고 경도에 대해 평균"을 내면 → 그 평균값은 위도의 main effect에 해당 → \(f_{\text{위도}}\)로 이동

    이 과정을 반복하면 \(f_{12}\)에는 두 변수가 동시에 특정 조합일 때만 나타나는 순수한 지역 효과(예: "SF 도심의 특정 블록")만 남고, 해안가 효과는 main effect로 돌아간다. 바다 위의 비정상적 패턴이 사라진다.

    사례 2: COMPAS (재범 예측)¶

    COMPAS는 미국 법원에서 사용하는 재범 위험 예측 시스템이다. Lengerich et al. (2020)은 이 데이터로 XGBoost depth-2를 학습한 뒤 shape function을 확인했다.

    무슨 일이 벌어졌나:

    특정 변수의 main effect가 purification 전후로 부호가 뒤바뀌었다:

    • Purification 전: main effect가 양(+) → "이 특성이 높을수록 재범 위험 증가"
    • Purification 후: main effect가 음(-) → "이 특성이 높을수록 재범 위험 감소"

    왜 이런 일이 가능한가:

    위의 숫자 예시에서 본 것과 같은 원리다. DTI의 main effect가 0이었는데 purification 후 -0.3/+0.3이 된 것처럼, 교호작용에 누출된 main effect의 크기가 원래 main effect보다 클 수 있다. 극단적으로는, 누출된 양이 원래 main effect의 부호를 뒤집을 만큼 커서 최종적으로 반대 부호가 되는 것이다.

    왜 위험한가:

    이 모형이 신용평가에 쓰인다고 하자. 규제 심사역이 "소득 변수의 효과는 단조적인가?"라고 물었을 때, purification을 안 한 shape function을 보여주면 틀린 답을 하게 된다. 알고리즘 공정성 감사에서도 "이 변수가 특정 집단에 불리하게 작용한다"는 판단이 뒤집힐 수 있다.

    모형별 Purification 영향¶

    모형 Interaction 처리 Purification 영향
    XGB depth-1 (= GAM) 없음 없음 — 교호작용 자체가 구조적으로 불가능
    XGB depth-2 이상 자동 생성, 분리 설계 없음 매우 큼 — main effect 부호까지 변할 수 있음
    GA\(^2\)M (EBM) 2-stage 분리 시도 중간 — 여전히 누출 존재 (아래 참조)

    GA2M(EBM)의 2-Stage 학습도 불완전하다¶

    EBM은 이 문제를 인지하고 2단계로 학습한다:

    1. 1단계: main effect만 학습 (변수 하나씩 round-robin boosting)
    2. 2단계: 잔차(residual)에서 교호작용 학습

    그럼에도 누출이 발생하는 이유:

    • 1단계에서 변수를 순환하며 학습할 때, \(X_1\)의 main effect가 \(X_2\)의 영향을 일부 흡수할 수 있다
    • 2단계에서 \(f_{12}\)를 학습할 때, 1단계에서 충분히 학습되지 않은 main effect가 \(f_{12}\)에 새어들어간다

    Lengerich et al. (2020)의 MIMIC-III 실험에서 이를 실증했다: BUN(혈중요소질소) 변수의 main effect가 purification 전후로 크게 변했다. 2-stage 학습이라는 ante-hoc 설계만으로는 main effect ↔ interaction 누출을 완전히 방지하지 못한다는 것이다.

    → 학습 단계의 분리(ante-hoc)만으로는 부족하고, 학습 후 purification(post-hoc 정제)이 필요하다. InterpretML의 EBM은 이를 인지하고, explain_global() 시 자동으로 purification을 적용한다.

    Purification은 fANOVA 제약을 트리 모형에 강제하여 순수한 main effect와 교호작용을 분리한다. 그렇다면 SHAP은 이 문제를 어떻게 처리하는가? 두 프레임워크의 근본적 차이를 다음 절에서 비교한다. → 4절. SHAP vs fANOVA — 무엇이 다른가
