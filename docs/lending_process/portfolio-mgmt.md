# ③ 사후 관리 — 실행 이후의 모니터링과 재평가

대출이 실행되면 만기까지 지속적으로 관리합니다.

## Behavioral Scoring (행동평점)

Application Score가 **신청 시점**의 정보로 평가한다면,
Behavioral Score는 **기존 거래 행동**을 기반으로 리스크를 재평가합니다.

- 입력 변수: 계좌 이용 패턴, 연체 이력, 결제 행동, 잔액 추이 등
- Application Score보다 예측력이 높은 경향 (정보량이 많으므로)
- 주기적으로 재산출 (월 1회 등)

모형 구조 자체는 가이드북에서 다룬 것과 크게 다르지 않습니다.
타겟 정의(Bad 정의)도 AS와 유사할 것으로 보이며,
핵심 차이는 **입력 변수**(신청 정보 vs 거래 행동)와 **대상 모집단**(AS: 신규 신청자, BS: 당행 여신 보유자)입니다.

## 한도 관리 (증액·감액)

Behavioral Score 기반으로 주기적 재심사를 수행합니다.
카드사의 경우 한도 증액은 매출(이자수익)과 직결되므로 적극적으로 운영합니다.

## 조기경보 시스템 (Early Warning System, EWS)

연체가 실제로 발생하기 전에, 위험 징후를 사전 탐지하는 시스템입니다.

- 시그널 예시: 급격한 카드 사용 증가, 현금서비스 비중 상승, 타 기관 연체
- 경보 발생 시: 선제적 접촉, 한도 축소, 모니터링 강화 등의 조치
- 모형 기반 EWS와 규칙 기반 EWS를 병행하는 것이 일반적

## Cross-sell / Up-sell

우량 고객에게 추가 상품(신용대출, 카드, 보험 등)을 제안합니다.
리스크 평가 + 반응 예측(Response Model)이 결합되는 영역으로,
여신 관리보다는 마케팅/CRM에 가깝습니다.
